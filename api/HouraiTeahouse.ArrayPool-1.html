Remarks

Renting and returning buffers with an ArrayPool<T> can increase performance in situations where arrays are created and destroyed frequently, resulting in significant memory pressure on the garbage collector.

This class is thread-safe. All members may be used by multiple threads concurrently.

Remarks

The shared pool provides a default implementation of ArrayPool<T> that's intended for general applicability. It maintains arrays of multiple sizes, and may hand back a larger array than was actually requested, but will never hand back a smaller array than was requested. Renting a buffer from it with Rent(Int32) will result in an existing buffer being taken from the pool if an appropriate buffer is available or in a new buffer being allocated if one is not available.

Remarks

This buffer is loaned to the caller and should be returned to the same pool via Return(T[], Boolean) so that it may be reused in subsequent usage of Rent(Int32).
It is not a fatal error to not return a rented buffer, but failure to do so may lead to decreased application performance, as the pool may need to create a new buffer to replace the one lost.

The buffer previously obtained from Rent(Int32) to return to the pool.
System.Boolean clearArray

If true and if the pool will store the buffer to enable subsequent reuse, Return(T[], Boolean) will clear array of its contents so that a subsequent consumer via Rent(Int32) will not see the previous consumer's content. If false or if the pool will release the buffer, the array's contents are left unchanged.
Remarks

Once a buffer has been returned to the pool, the caller gives up all ownership of the buffer and must not use it. The reference returned from a given call to Rent(Int32) must only be returned via Return(T[], Boolean) once. The default ArrayPool<T> may hold onto the returned buffer in order to rent it again, or it may release the returned buffer if it's determined that the pool already has enough buffers stored.
